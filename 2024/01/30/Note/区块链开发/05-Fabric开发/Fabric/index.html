Fabric

简介

​ Fabric是一个由模块化架构支撑的分布式账本解决方案平台，提供高度的保密性、弹性、灵活性和伸缩性。它被设计支持不同组件的插拔，并且适应经济生态系统的复杂性。

​ Fabric是一个开源的企业级需要许可的分布式账本技术平台，由Hyperledger开发。与比特币、以太坊等无许可的公链不同，Fabric是一个有许可的联盟链。

​ Fabric由以下模块化组件组成：一个可插拔的排序服务用于建立共识并广播区块给对等方；一个可插拔的成员资格服务提供商负责将网络中的实体与加密身份相关联；一个可选的点对点八卦服务用于向其他对等点订购服务并传播块输出；智能合约（链码）在容器环境（例如 Docker）中运行以进行隔离，并可以用标准编程语言编写；分类帐可以配置为支持各种DBMS；可插入的背书和验证策略实施，可以为每个应用程序独立配置。

文档：https://hyperledger-fabric.readthedocs.io/zh-cn/latest/getting_started.html

目录结构

main

  • bccsp：加密标准以及算法实现
  • bddtests：bdd测试框架相关的测试
  • common：通用功能以及一些通用的代码实现
  • core：核心代码
  • discovery：服务发现
  • doce：文档
  • event：提供事件订阅
  • examples：案例
  • gossip：信息传播模块
  • ldemix：零知识证明
  • integration：代码集成
  • gotools：用于编译fabric
  • msp：提高成员服务
  • orderer：排序节点
  • peer：peer节点
  • proposals：存放相关提案
  • protos：存放各种协议和消息的protobuf定义文件和生成的go文件
  • release_notes：各个版本的更新日志
  • sampleconfig：相关样例配置i文件
  • scripts：存放相关脚本文件
  • unit-test：单元测试
  • swagger：接口文档
  • tools：工具
  • vagrant：创建虚拟机，自动化配置和安装开发环境

模块

  • 成员管理
    • 成员注册：获得一个证书用于身份认证
    • 身份保护：
    • 交易审计：

角色

  • 组织
    • 可以有多个，备份同组织全部信息，避免单点故障
    • 组织下有用户节点，处理数据的节点peer
  • 节点
    • 客户端：进行交易管理，可以通过shell、go、nodejs、java等实现
    • peer：存储和同步账本数据，fabric框架实现
    • orderer：排序节点，主要是排序和分发交易，打包写入区块链，排序解决双花问题，排序节点可复用
  • 通道
    • 将peer节点和orderer节点链接在一起，形成一个具有保密性的虚拟通讯链路，实现业务隔离
    • 通道也和账本、状态紧密相关
    • 一个peer可以加入不同的通道
    • 不同通道使用不同的区块链
  • 账本
    • 存储了交易记录、世界状态、数据库

    2. 生成证书 cryptogen generate --config conf.yaml

    2. 生成创世区块
