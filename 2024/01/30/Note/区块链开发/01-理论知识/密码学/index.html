密码学

模型

椭圆曲线

  • 是一个二维平面上点的集合，这些在x轴上对称

  • P + Q = R（PQ连线交于-R，映射到R即为结果）

  • K = k * G => G1+G2+…Gk
    • 已知K和G，无法求出k

SHA哈希算法

SHA1

SHA256

  • collisiom resistance：输入转hash结果不变，但存在哈希碰撞
  • hiding：根据哈希值无法预测输入
  • puzzle friendly：根据输入无法预测哈希值

SHA-512

HMAC-SHA512

  • 键控哈希算法，结合了密钥和SHA-512哈希函数来提供数据的完整性和认证
  • HMAC代表哈希消息认证码（Hash-based Message Authentication Code），它是一种通过特定算法使用密钥和消息作为输入来生成消息摘要的安全机制
  • 过程
    • HMAC进程将密钥与消息数据通过特定的方式组合，并对其进行哈希处理
    • 这个哈希值再次与密钥混合，并进行第二次哈希运算。最终输出的哈希值长度为512位

签名算法

ECDSA

  • 基于椭圆曲线的签名算法，不支持聚合签名，多签需要逐个验证
  • 计算出r, s并返回用于rsv签名
    • v是一个恢复ID。v的值是由r的奇偶性决定的，如果r是偶数，则v=27；如果r是奇数，则v=28
  • 计算出私钥k，公钥K
  • 过程
    1. 选择一条椭圆曲线Ep(a, b)，和基点G（对应曲线的G值是确定的）。
    2. 获取私钥k（k是利用随机数生成器生成的一个256位的二进制数字），再利用基点G计算公钥K = k * G
    3. 产生一个随机整数r，计算点R = r * G
    4. 获得R的坐标x，y
    5. 计算HASH = SHA1(原数据, x, y)
    6. 计算s = r - Hash * k (mod n)
    7. r和s做为签名值。如果r和s其中一个为0，重新从第3步开始执行
  • 签名 = Hash（私钥k，消息，r，s，v）

BLS

  • 可将多个签名聚合成一个签名，并且可以对每个签名进行验证，不需要与签名者交互

  • 准备

    1. 准备一个椭圆曲线：y²=x³+ax+b
    2. 准备一个曲线配对函数，需要满足以下特性
      • e(P, Q) → n （P、Q是椭圆曲线上的两个点）
      • e(x * P, Q) = e(P, x *Q) （x是未知数）
      • e(a*P, b*Q) = e(P, ab*Q) = e(ab*P, Q) = e(P, Q)^(ab)^

  • 签名

    • 将消息m经过哈希函数 H() 算出一段哈希值 H(m)，映射到椭圆曲线上的x轴，获得两个、一个或0个对应的y值

      • 取得两个值时取较小的一个
      • 如取不到两个值的话，在原有消息之后添加一个数再进行哈希求解，直到取到两个值为止

    • 选一个私钥pk，公钥P = 私钥pk * 椭圆曲线基点G

    • 签名S = 私钥pk * 加密信息H(m)

    • 验证签名

      • e(公钥P, 加密信息H(m)) = e(私钥pk * 椭圆曲线基点G, 加密信息H(m)) 
                     = e(椭圆曲线基点G, 签名S) 
                     = e(椭圆曲线基点G, 签名S)

    • 验证时对所有的公钥进行验证即可，减少了存储空间和验证时间

Schnorr

  • 签名和公钥较短，支持聚合签名，但需要与签名者进行交互

Ed25519

  • Ed25519是一种计算快，安全性高，且生成的签名内容小的一种不对称加密算法。新一代公链几乎都支持这个算法

RIPEMD-160信息摘要算法

​ RIPEMD-160（RACE Integrity Primitives Evaluation Message Digest）是一种信息摘要算法，主要用于数字签名和认证，也是区块链中使用的信息摘要算法之一。RIPEMD-160是由荷兰FOM（Institute for Applied Mathematics St. Martens Voeren）以及比利时UCL（Universite catholique de Louvain）在1992年推出的信息摘要算法，这种算法由160位的消息摘要组成，具有更高的安全性和流行性。

​ RIPEMD-160是一个修改版本的MD4算法，能够生成20字节的摘要值，安全性基本等同于SHA-1（注意，SHA-1标准没有说K值是怎么确定的，这一点非常可疑）。RIPEMD-160可视为左右两个MD4并行计算，如果使用两个线程，性能不会有很大区别。RIPEMD-160内部维护64字节的输入缓存，20字节的输出缓存、20字节左计算缓存、20字节右计算缓存、8字节计数器和1字节的输入计数器。

​ RIPEMD-160的一个典型应用是在比特币和其他基于比特币的加密货币中，它被用来生成公钥哈希（public key hash），即地址（address）。具体来说，比特币地址是由以下步骤生成的：

  1. 生成一个椭圆曲线数字签名算法（ECDSA）的公钥和私钥对。
  2. 对公钥进行SHA-256哈希运算，得到一个256位的哈希值。
  3. 对上一步得到的哈希值进行RIPEMD-160哈希运算，得到一个160位的哈希值，即公钥哈希。
