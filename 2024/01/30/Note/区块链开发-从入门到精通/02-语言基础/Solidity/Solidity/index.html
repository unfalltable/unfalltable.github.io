Solidity

简介

  • Solidity是一门面向对象的，强类型的高级编程语言，用于智能合约开发

  • Solidity是单线程的，大部分情况下是线程安全的

  • 格式

数据类型

  • 状态变量：写在合约内函数外，如果没有提供修改的方法，那么将永远写在区块链上，默认是internal
  • 局部变量：写在函数内，调用函数时就会在虚拟机中产生，不能使用public
  • 全局变量：内置变量

基本数据类型

byte

  • 字节，是固定长度的数据类型，一般是以16进制的形式展示的
  • 有 bytes1~32 字节的定长数组
  • 默认值：0x0000…

int

  • 整型，默认是256位
  • 有 int1~256 位，uint是无符号整型
  • int256(-1)是整型最大值
  • 默认值：0

bool

  • 布尔，用来表示真或假
  • 默认值： false

address

  • 地址，在以太坊中存储的是20字节的十六进制格式的值，即uint160
  • 默认值：0x0
  • address有成员变量，用payable修饰地址会有 transfer 和 send 成员变量

enum

  • 枚举
  • 取值是取下标
  • 默认值：第一个值

Function

  • 函数类型也被归类为基本数据类型

string

  • 字符串，底层是字节数组，消耗的gas多
  • 默认值： “”

bytes

  • 字节数组，bytes的gas更便宜
  • 默认值： []

fixed / ufixed

  • 定点数，用于表示固定位数的小数
  • fixed16x8 表示一个总共32位的整数，其中高16位是小数部分，低16位是整数部分
    • 123456789012345678 等同于 123.456789012345678

引用数据类型

mapping

  • 映射，底层是hash表，键值存储，键key不能是结构体
  • 不支持遍历
  • 默认值：0

数组

  • 类型[]
  • 默认值：类型默认值

struct

  • 结构体
  • 创建对象不用new
  • 默认值：成员默认值

关键字

关键字 作用 说明
constant 定义常量 gas费较低，不能修改
immutable 将变量定义为常量 gas费较低，需要赋值
indexed 事件中用于修饰变量，他们会保存在以太坊虚拟机日志的 topics 中 相当于检索事件的索引“键”，方便搜索，大小为固定的256比特，超过这个大小会自动计算成哈希存储
ether 以太坊代币单位 1 ether = 10^18 wei
memory 将变量存储在内存 只在函数调用期间存在，存储在EVM堆内存中，函数执行结束就会被清除，消耗gas少
Storage 变量会永久存储在区块链 函数之外声明的变量默认为storage类型，消耗gas多
calldata 将变量存储在函数之外 只能修饰输入参数，不能修改，函数间传递参数消耗gas少，存储在合约的ABI编码中
public 公共的 修饰的状态变量会生成访问函数，修饰的函数可以被调用，内外部都可以调用
private 私有的 合约内部可用，不可被继承，比public节约gas
external 外部的 只能通过外部读取函数，可以通过this调用但消耗gas，不可被继承
internal 内部的 合约内部可用，可以被继承
payable 付费 调用合约方法需要转账，标记的地址可以发送代币
view 可视的 可以读取状态变量，不能修改状态变量
pure 纯函数 不能读取状态变量，只能拥有局部变量，用户调用pure函数不用gas，合约调用需要支付gas
virtual 可重写的
override 重写的 修饰 public 变量会重写该变量的 get 方法
returns 返回值
unchecked 不进行溢出检查 提高执行xiao’lü

合约

简介

  • 智能合约一般是用来编写控制逻辑的

  • 智能合约编译后的字节码最大不超过24KB

  • 发布到区块链上的交易不一定都是成功执行的

    • 为了扣除汽油费

  • 智能合约可以获得的信息比较有限

    • 因为每个节点的环境不同

进阶

Gas优化

  • 每5个byte需要1个gas
  • 尽量使用常量，constan，immutable
  • 选择合适的数据类型，尽量使用int256而不是int8等
    • 因为使用int8等，没使用到的空间EVM会用0填充，计算时也会转换为int256
  • 尽量使用批量操作，因为一笔基础交易的gas是21000，批量操作可以减少多次交易的基础gas消耗
  • 循环中避免不必要的操作
  • 使用最新版本的solidity编译器，一般会有更好的优化和性能改进
  • 状态变量复用，而不每次都从链上读取
    • 频繁修改时可以先记录修改，最后在一次性写回状态变量
  • 结构体中如果有多个int类型，尽可能使用小的int，因为这些int会紧密打包在一起，从而占用更小的存储空间
  • 能使用默克尔树替换mapping或数组时则优先使用默克尔树
  • 考虑将复杂的计算在链下进行，将结果存储在链上
  • 使用第三方库而不是自己造轮子
  • 用++替换+=
  • 循环判断条件使用 && 拼接
  • 当函数代码太多时可以将一些代码用单独的大括号划分出来，避免堆栈太深导致gas超额

值传递还是引用传递

  • 函数外的storage状态变量赋值给函数内的storage变量时，修改函数内storage变量会影响函数外的状态变量
  • 函数外的storage状态变量赋值给函数内的memory变量时，修改函数内memory变量不会影响函数外的状态变量
  • memory变量赋值给memory变量时，会影响原变量
  • 变量赋值给storage变量，会创建独立的副本
