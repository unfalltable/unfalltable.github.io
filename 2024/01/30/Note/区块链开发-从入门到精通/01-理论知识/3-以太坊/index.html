以太坊

简介

​ 以太坊是一个基于区块链技术的去中心化应用平台，允许开发者在其上创建和发布去中心化应用程序。以太坊的概念最初由程序员维塔利克·布特林（Vitalik Buterin）在2013至2014年间提出，其核心思想是提供一个类似于比特币的去中心化加密货币平台，但具有更强的灵活性和可扩展性。

​ 以太坊的运行是基于EVM虚拟机，运行的是去中心化的智能合约，发布后不可修改。与比特币不同的是，以太坊是由余额的性质的，这样能有效的避免双花攻击，余额是通过状态树来维护的，避免余额被篡改。

​ 以太坊有自己的代币Ether，1Ether = 10^18^Wei；区块的大小通过燃油费来限制，最多不能超过3000万单位燃料，防止代码太多导致区块很大，矿工计算时更加耗时费力，还能防止死循环；以太坊中两个区块之间发布的时间间隔以时隙为单位，每12秒为一个时隙

账户

  • 以太坊1.0中使用的是和比特币一样的POW共识机制，使用的是椭圆曲线数字签名算法ECDSA中的secp256k1曲线生成的公私钥对，不同的是公钥转换为地址的方式。
    • 将公钥K的坐标x和y拼接在一起，然后通过Keccak256算法，然后截取低位20字节的值用作地址
  • 以太坊中账户是可以存储信息的，如果存储了智能合约代码，那么这个账户就是一个智能合约账户。
  • 账户中通过一个随机数来记录账户的交易次数
  • 账户地址160位，40个16进制的数

结构

节点

  • 轻节点
    • 保存一个16m的Cache
  • 全节点
    • 保存一个16m的Cache和1G的Dataset

区块结构

  • 区块
    • 所有与账户相关的活动都会以交易的格式存储在区块中，每个区块中都有一个交易列表
    • 交易执行结构，日志记录
    • 不同区块之间通过ParentHash进行链接，哈希是通过Keccak256计算的
  • 区块体
    • 详细的交易列表信息
    • 叔父区块的信息

状态树

  • Modified Merkle Patricia Tree

  • 存储的是账户地址到账户状态的映射，键值对

    • key是地址
    • value是账户的状态，通过RLP进行序列化

  • 状态树的结构类似一个字典树，并对节点进行了路径压缩，相同前缀的地址存储在同一颗子树上

    • 每个账户地址都是唯一的，且查找方便
    • 因为生成地址的哈希算法符合泊松分布，随机性好比较稀疏，所以适合路径压缩
    • 路径压缩可以有效的降低树的高度

  • 每次发布区块时，没有发生交易的地址在区块之间时共享的，参与交易的地址，需要新建分支

    • 维护旧地址是为了保留历史状态，因为交易可能回滚
    • 因为智能合约的设计复杂，无法推断出账户之前的状态，所以需要保留账户的历史信息

交易树

  • 结构和状态树一样
  • 记录的是交易信息

收据树

  • 结构和状态树一样
  • 结合布隆过滤器一起使用
  • 每次交易
  • 记录的是交易的收据信息，记录了交易的执行结果，收据列表的长度和交易列表的长度一致

挖矿

  • 早期的以太坊使用的是基于Ghost协议的POW工作量证明共识机制，平均出块时间大约12秒，由于出块时间短，导致出现多个节点同时出块的可能性很高，所以临时性分叉很多

  • 发布新的区块时不会包含叔父区块中的交易，会检查叔父区块的合法性（是否符合挖矿难度要求）

  • 只有分叉后的第一个叔父节点可以得到奖励

  • 矿池

    • 由许多矿工组成，一起挖矿并按劳分配奖励
    • 大型矿池更有可能成为最长合法连，马太效应

  • 奖励

    • 叔父区块可以获得出块奖励，每次发布新区块最多只有两个叔父区块可以获得奖励
      • 七代以内的叔父区块才有奖励，逐代递减1/8
    • 发布区块包含叔父区块可以获得1/32奖励，包含多个就有多份奖励

步骤

  1. 通过一个种子seed通过某种运算算出一个值，然后依次经过这个运算取值存入cache数组
    • 每隔30000个区块会重新生成seed，即对原来的seed取hash
    • cache数组的初始大小是16M，每隔30000个区块会增大128k
  2. 生成一个更大的dataset数组
    • 初始大小是1G，每隔30000个区块会增大8M
    • 读取cache数组第 i % cache_size ，对这个数 异或 i 再取hash，记为mix
    • 循环256次
      • 通过函数传入mix算出下一个读取的下标
      • 通过函数传入mix和 cache[下标 % cache_size]，得到新的mix
    • 得到 hash(mix) 存入dataset[i]
    • 循环这个操作直到填满dataset
  3. 根据块头和nonce值，算出一个初始的hash，映射到dataset数组的某个位置和他的下一个位置
    • 全节点直接访问内存，轻节点通过下标间接访问
  4. 然后这两个位置的数在进行运算得出下两个位置，循环这个操作64次
  5. 最后算出的值和挖矿难度的目标阈值比对，满足则通过，不通过则改nonce

难度调整

  • 难度为：max（131072, 父区块难度 + X * Y） + Z
    • 最低难度：131072
    • X：难度调整单位，上一个区块的难度的2048分之1
    • Y：max（n - m，-99）
      • n：有uncle区块为2，否则为1
      • m：（当前区块的时间戳 - 父区块的时间戳） / 9，向下取整
    • Z：2^(区块号-3000000)/100000-2^
      • 难度炸弹，挖矿初期难度炸弹影响较小，越往后难度指数增加

交易

  • 以太坊中每笔交易都要付燃油费gas，每5个byte需要1个gas
  • 早期以太坊使用的签名算法和比特币中的一样，是ECDSA算法，但是随着验证者的数量越来越大，交易越来越多，验证签名的时间达到了30分钟以上，无法出块。而且每个区块的签名量很多，会缓存很多的签名，带来了存储问题。所以在以太坊2.0改为POS权益证明共识机制并将签名算法改为BLS
