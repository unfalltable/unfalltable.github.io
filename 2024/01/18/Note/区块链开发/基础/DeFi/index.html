DeFi

简介

  • 去中心化的金融交易所
    • 中心化金融CeFi：受相关部门监管，严格遵守规定，客户没有隐私，信息不透明，客户权限有限
    • 去中心化金融DeFi：资产由客户自己掌控，无监管，信任、低门槛、高效、高隐私性、可组合、可交互、透明、自动化、可编程、开发简单
  • 可以链上借贷，闪电贷，超额抵押，快速套利（价格差）
  • 提供稳定币，通过链下抵押资产、链上抵押资产、算法实现（波动较大）实现

闪电网络

简介

  • 链上开启一个支付渠道，交易在链下记录，一定时间后清算链下的交易，并提交到链上执行

Layer2

简介

  • 区块链2层网络
  • 跨层的交易手续费比较高，但是同层的交易则比较低廉

Rollup

  • 在以太坊Layer1上部署合约
  • 时刻更新交易数据到Layer2网络中
  • 将交易按批次打包交易提交到Layer1的智能合约上
    • 乐观打包
      • Optimistic
        • 引入争议时间延迟机制
        • 使用挑战期保护数据正确性
        • 容易实现
      • Arbitrum
        • 多轮交互式欺诈证明
        • 可组合性
    • 零知识证明打包
      • 安全性依赖密码学，不需要挑战期
      • 计算能力较差
      • 需要通过协调者来验证交易的合法性
      • 协调者
        • 当协调者停止工作时，需要新的协调者，新协调者需要获取当前所有用户的信息
          • zkSync
            • 将所有交易摘要信息存储在L1区块链上
            • 新的协调者可以根据这些信息重构L2的状态
            • 缺点是交易费用高
          • zkPorter
            • 将交易数据存储在一个新的区块链上
            • 由一组协调者共同维护
            • 廉价的链外存储
  • 然后以太坊主网验证

跨链

  • 将一条链的资产转移到另一条链上
  • 一般通过两条链的智能合约实现
    • 一般通过资产冻结和生成实现

交易所DEX

架构

  • 用户发布需求
  • 交易所撮合交易
  • 交易所交易结算

订单簿

  • 中心化
    • 快速撮合
    • 取消订单无需手续费
    • 不抗审查
    • 交易所抢先交易
  • 取中心化
    • 抗审查
    • 鲁棒性、稳健
    • 撮合缓慢
    • 每一步都需要手续费
    • 矿工 / 交易者抢先交易

自动化做市商AMM

简介

  • 让智能合约来做市商
  • 创造一个流动性池
  • 公式：X * Y = K

性质

  • 即时流动性
  • 购买其中X资产会让X价值提高，Y价值降低
  • X和Y的资产量决定了价格：Y = P * X

滑点

  • 就是在发生交易时反比例函数中对应的点移动的区间

  • 我们希望的是这个滑点区间接近切线的位置

  • 滑点区间大意味着用户置换货币的比例是比较糟糕的

优缺点

  • 无订单簿维护
  • 较低的gas花费
  • 无常损失，币价脱钩的风险，资金可能全部流失
  • 流动性低，滑点较高
  • 容易受到三明治攻击

兑换交易传播

  • 是异步区块链P2P网络
    • 尽最大努力传播
    • 透明、高频交易
  • 是基于费用的拍卖
    • gas价格拍卖
    • gas密封投标拍卖
    • gas越高交易越快执行

套利

  • 利用多个交易所的价格差异实现套利
  • 可以通过代币置换或者闪电贷套利
  • 工具
    • 贝尔曼福特算法 ARB
      • 建立Defi市场图标
      • 通过负循环检测
      • 贝尔曼福特算法计算
    • 定理求解器 SMT
      • 需要对DeFi模型编码
      • 需要一些启发式算法来进行路径修剪
      • 定理证明者

流动性挖矿

简介

  • 提供流动性可以获得流动性挖矿奖励，存在获得空投的可能性
  • 可以随时添加和移除流动性
  • 可以获得交易手续费

无常损失

  • 是AMM中最大的风险，损失资产最大的地方
  • 流通性提供者会因为流动性池中的代币上涨或下跌可能会导致无常损失
  • 套利者会利用代币上涨来获取更多的代币以获得利益
  • 无常损失可能会导致全损

聚合器

  • 获得更好的交易价格
  • 获得最深的流动性位置
  • 有两种类型的聚合器
    • 链外聚合器
      • 能生成多链，UI简洁
      • 由操作员发布信息
    • 链上聚合器
      • 原子性路径选择
      • 一般覆盖不了4个以上的交易所

借贷

简介

  • 放款方：存入资金 / 赎回资金 + 利息
  • 借款方：抵押 / 借款
  • 清算者：清算
  • 价格预言机：更新价格
  • 超额抵押
    • 抵押资产需要大于借款金额
  • 抵押不足
    • 抵押资产需要小于于借款金额
    • 借款只能用于特定的用途
  • 清算：
    • 任何人都可以进行清算并获取奖励
    • 清算人偿还待清算用户的债务来获取抵押物
    • 连续多次的清算可能可以获得更高的收益
    • 清算情况
      • 抵押物价值 <= 借款 * 150% 即可清算
    • 清算方式
      • 清算拍卖：多个清算人随时间出价
      • 即时清算：在一次交易中进行清算
      • 抵押物闪贷：不偿还债务直接清算
      • 去杠杆螺旋清算：清算后降低抵押物价格再进行清算
      • 超额清算：出售过多的抵押物
  • 健康因子
    • 健康因子 = 抵押物价值 * 清 算阈值 / 债务价值合计
    • 0 < 清算阈值 < 1
    • 健康因子小于1即可清算
  • 清算价差
    • 清算者在清算抵押物时可以获得的奖励
    • 一个比例
  • 关闭因子
    • 在一固定价差清算中允许偿还的最大债务比例

闪电贷

  • 由闪电贷供应商提供流动性
  • 可以提升资本利用效率和为套利者扳平价格
  • 容易被攻击者利用
  • 借贷换贷都在一次交易中完成
  • 用途
    • DeFi攻击
      • 价格预言机操纵
      • 拉高出货
    • 套利
    • 倒仓
    • 闪铸
    • 抵押物替换

稳定币

简介

  • 相对其他加密货币更稳定

    • 稳定的价格 / 购买力

  • 抵押稳定币

    • usdt、usdc、dai

  • 算法稳定币

    • AMPL：通过价格预言机实现通胀增发，紧缩销毁

  • 种类

    • 托管

      • 风险：交易对手信用风险、审查风险、传统金融风险
      • 方式：储备基金、中央银行、银行基金、货币市场基金

    • 非托管

      • 风险：去杠杆风险、喂价风险、治理风险、矿工提取价值、只能合约bug
      • 方式：外源性抵押品、内源性抵押品、隐形抵押

  • 风险容灾

    • 基于杠杆：增加更多的抵押品，增加铸币税
    • 基础设计：在危机时期进行押注
    • 储备支持：使用内部储备

安全

  • 预言机攻击
  • MEV：分叉模型
  • GEV模型

衍生品

期货合约

  • 约定在未来的某一天以一个价格购买现货，不管价格是上涨还是下跌
  • 没有对手风险
  • 提供保证金，如果账户保证金过少需要追加
  • 每日根据期货市场价格的收益和损失都会在账户显示
  • 保证有一个共同的期货市场价格
  • 在生产者和投机者之间交换风险
  • 交易所对抵押物进行管理，交易者可以安全的进行交易

远期合约

  • 约定在未来的某一天以某个价格购买一定数量的现货的承诺
  • 定制化，没有资金还手，有很大的对手风险

掉期合约

  • 大机构和国际掉期与衍生工具协会监管ISDA签订主合约
  • 固定收益和浮动收益掉期
    • 只交换现金流
    • 本金没有交换
  • 信用违期掉期
    • 类似保险
    • 买家按期支付利息给卖家
  • 股权回报掉期
  • 掉期合约为什么这样设计
    • 交易者想直接进行交易但是过于昂贵
    • 只交换净现金流，还能降低交易对手风险
    • 能够灵活的构建合成资产
  • 中心化机构清算
    • 净交易
    • 双边结算的顺序导致低效率

合成资产

  • 将链下的资产映射到链上
    • 如可以购买黄金，法币等
  • 前提
    • 识别可能的风险
    • 衡量风险变化的稳健方法
    • 如果没人拥有真正的合成资产，如何确保支付
    • 如何使新资产易于交易
  • 使用
    • 超额抵押SNX
    • 发行sUSD债务
    • 使用sUSD购买任何东西
    • 偿还债务，赎回SNX
  • 动态交易池

期权合约

永续合约

  • 最大限度的提高流动性
  • 通过杠杆提高本金
  • 需要提交初始保证金和追加保证金

对冲基金

  • 高杠杆、高净值、税收优化

私人产权

  • 仅在项目成立做
  • 有限合伙人出资
  • 通常高杠杆
  • 非流动性投资

Uniswap

  • 提供代币质押兑换，需要付手续费

  • 发行代币不用经过审查，只需要满足其代币标准和流通性标准即可

  • 代币置换规则（CPAMM - 恒定乘积做市商）

    • 规则：A币 * B币 = 常数
    • 例子：
      • 假设 1A = 1000B，池子中有 10A，10000B，则常数为100,000
      • 当用户想出售一个A换B时，池子中
        • A = 10 + 1 = 11
        • B = 100,000 / 11 ≈ 9090
        • 即置换出 10000 - 9090 = 910B
        • 常数 = 11 * 9090 = 99,990
      • 置换比例不是 1：1000的比例，是因为例子中币数量太少，导致兑换比例差距过大
      • 大池子中这个比例就会很接近 1：1000的比例

  • V3和V2的区别

    • 流动性提供方式：在Uniswap V2中，流动性提供者需要预先将资产锁定在合约中，这使得他们无法在合约之外使用这些资产。而在Uniswap V3中，流动性提供者可以通过智能合约将资产自由地借出或归还，而不需要锁定它们。这使得流动性提供者的操作更加灵活。

    • 交易逻辑：Uniswap V3和V2的交易逻辑也有很大的不同。在V2中，交易是直接在AMM池中进行的，而V3则引入了一个新的交易类型——单币交易。这意味着在V3中，用户可以单独对某一个币进行交易，而不需要考虑AMM池中的其他币种。

    • 费用结构：在费用结构上，V3也进行了优化。它引入了两种新的费用类型：交易费和滑点费。交易费是针对每笔交易收取的费用，而滑点费则是针对交易对价格波动收取的费用。这种费用结构可以更好地激励流动性提供者提供更好的交易体验。

    • 虚拟流动性：为了提高资金利用率，但是只能在特定的价格区间做市，需要用户自己设置需要做市的价格区间

      • v2公式
        • x * y = k（x是代币A的数量，y是代币B的数量，k是流动性代币的数量）
        • y = p * x（p是价格）
        • p * x^2^ = k
        • x = $\sqrt{k / p}$，y = $\sqrt{k * p}$
      • v3公式：
        • （x + x_virtual) * （y + y_virtual）= k
        • x_virtual = $\sqrt{k / p(最高价)}$ ，y_virtual = $\sqrt{k * p(最低价)}$
